## Architecture Overview

The Dark Lion Custom Uniforms application is built with React 19 and uses a component-based architecture focused on customization workflows and 3D visualization.

## State Management

The application uses **React hooks** for state management without external state libraries.

### Local Component State

Most components use `useState` for internal state:

```jsx
const [fabricType, setFabricType] = useState("");
const [design, setDesign] = useState("");
const [colors, setColors] = useState({ torso: "", back: "", sleeves: "", collar: "" });
```

### Navigation State

Design data is passed between routes using React Router's location state:

```jsx
navigate("/datos-seleccionados", { 
  state: { design: designData, players, modelo } 
});
```

### State Persistence

State is restored when navigating back:

```jsx
useEffect(() => {
  if (locationState?.design) {
    setFabricType(design.fabricType || "");
    setDesign(design.design || "");
    // ... restore other fields
  }
}, [locationState]);
```

<Note>
The application does not use Redux, Zustand, or other state management libraries. All state is local or passed via navigation.
</Note>

## Design Patterns

### Compound Components

The `Sidebar` component uses compound components for modular sections:

```jsx
<Section title="Cuerpo" icon={Square3Stack3DIcon}>
  <ColorPalette
    selected={colors.torso}
    onSelect={(c) => setColors((p) => ({ ...p, torso: c }))}
  />
</Section>
```

### Controlled Components

All form inputs are controlled components:

```jsx
<input
  type="text"
  value={teamName}
  onChange={(e) => setTeamName(e.target.value)}
  placeholder="Equipo..."
/>
```

### Prop Drilling

Player data is passed down through props:

```jsx
<PlayersSection players={players} setPlayers={setPlayers} />
```

<Warning>
For larger applications, consider implementing Context API or a state management library to avoid deep prop drilling.
</Warning>

## Styling Approach

### Tailwind Utilities

Primary styling method using utility classes:

```jsx
className="w-full bg-personalizado py-3 rounded-lg text-lg shadow-sm
          hover:scale-110 transition-transform duration-200"
```

### Custom Theme Colors

Custom color variables in Tailwind config:

- `bg-personalizado` - Primary background
- `bg-personalizado-click` - Interactive elements
- `bg-personalizado-click-hover` - Hover states

### Custom Fonts

Inline font-family styles:

```jsx
style={{ fontFamily: "Montserrat1" }}
style={{ fontFamily: "Arboria" }}
```

### RGB Color Values

Colors stored as RGB strings:

```jsx
"rgb(255, 255, 255)"
"rgb(178, 34, 34)"
```

## Routing Structure

The application uses these key routes:

- `/personalizar/:modelo` - Main customization page with `Sidebar`
- `/datos-seleccionados` - Design summary with `SelectedDataSidebar`
- `/agregar-logo` - Logo placement editor

## Component Communication

### Parent to Child

Props pass data down:

```jsx
<ContactForm
  data={contactData}
  onChange={handleChange}
  onSubmit={handleSubmit}
  fieldErrors={errors}
/>
```

### Child to Parent

Callback functions pass data up:

```jsx
const handleChange = (field, value) => {
  setContactData(prev => ({ ...prev, [field]: value }));
};
```

### Navigation-Based

Location state for cross-route data:

```jsx
const { state: locationState } = useLocation();
```
