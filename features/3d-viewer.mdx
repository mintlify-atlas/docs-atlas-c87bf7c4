## Overview

The 3D Uniform Viewer is a real-time visualization component that displays your custom uniform design in an interactive 3D environment. Built with Three.js and React Three Fiber, it provides an immersive preview experience that allows users to inspect their design from every angle.

<Note>
  The 3D viewer automatically loads when you enter the customization page and updates in real-time as you modify your design.
</Note>

## Core Features

### 1. 3D Model Display

The viewer loads and renders a 3D uniform model in GLB format:

```javascript
function ShirtModel() {
  const { scene } = useGLTF("/camisa3D/camiseta.glb");
  return <primitive object={scene} scale={1.5} />;
}
```

**Key Details**:
- **Model Format**: GLB (Binary glTF)
- **Model Path**: `/camisa3D/camiseta.glb`
- **Scale**: 1.5x for optimal viewing size
- **Loading**: Uses `useGLTF` hook from `@react-three/drei`

**Code Reference**: `src/components/ShirtViewer3D.jsx:5-8`

<Tip>
  GLB format provides efficient loading and rendering performance for web-based 3D models. The model is cached after initial load for faster subsequent renders.
</Tip>

## Lighting Setup

The scene uses a two-light setup for optimal model visibility:

### Ambient Light

```javascript
<ambientLight intensity={0.5} />
```

- **Type**: Uniform ambient illumination
- **Intensity**: 0.5 (50% brightness)
- **Purpose**: Provides base lighting to prevent harsh shadows
- **Color**: White (default)

**Code Reference**: `src/components/ShirtViewer3D.jsx:14`

### Directional Light

```javascript
<directionalLight position={[10, 10, 5]} intensity={1} />
```

- **Type**: Directional (sun-like) light
- **Position**: [X: 10, Y: 10, Z: 5] (top-right-front)
- **Intensity**: 1.0 (100% brightness)
- **Purpose**: Creates depth and highlights uniform details
- **Shadows**: Can be enabled for enhanced realism

**Code Reference**: `src/components/ShirtViewer3D.jsx:15`

## Performance Optimization

### Suspense Loading

The component uses React Suspense for efficient model loading:

```javascript
<Suspense fallback={null}>
  <ShirtModel />
</Suspense>
```

**Benefits**:
- **Non-blocking**: UI remains responsive during model load
- **Graceful handling**: No flash of loading state (fallback={null})
- **Error boundary**: Prevents crashes if model fails to load
- **Progressive loading**: Model appears when ready

**Code Reference**: `src/components/ShirtViewer3D.jsx:16-18`

<Tip>
  The 3D model is cached by the browser after the first load, making subsequent visits nearly instantaneous.
</Tip>

### Model Optimization

The GLB format provides several performance advantages:

- **Binary format**: Faster parsing than JSON-based formats
- **Compressed**: Smaller file size for quicker downloads
- **Embedded textures**: Single file contains all assets
- **GPU-ready**: Optimized for WebGL rendering

## Troubleshooting

### Common Issues

<Accordion title="Model not loading">
  **Symptoms**: Blank 3D viewer area
  
  **Solutions**:
  - Check that `/camisa3D/camiseta.glb` exists in the public directory
  - Verify network requests in browser DevTools
  - Ensure proper GLB file format
  - Check for CORS issues
</Accordion>

<Accordion title="Performance issues">
  **Symptoms**: Laggy rotation or controls
  
  **Solutions**:
  - Enable hardware acceleration in browser settings
  - Close other resource-intensive tabs
  - Check GPU compatibility
  - Reduce model complexity if possible
</Accordion>

<Accordion title="WebGL errors">
  **Symptoms**: Error messages about WebGL context
  
  **Solutions**:
  - Update graphics drivers
  - Try a different browser
  - Check if WebGL is enabled in browser flags
  - Test on [webglreport.com](https://webglreport.com)
</Accordion>
