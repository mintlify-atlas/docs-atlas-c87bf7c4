## Overview

The Team Management feature allows esports teams to build complete rosters with detailed player information and uniform specifications. Each player entry includes personal details, size preferences, jersey numbers, and quantity requirements, providing a comprehensive order management system.

<Note>
  Team rosters are saved along with your uniform design and can be edited at any time before final submission.
</Note>

## Interactive Features

### 1. Adding Players

Add new team members with the green "+" button:

```javascript
const handleAdd = () => {
  const newPlayer = {
    id: Date.now(),
    name: "",
    gender: "",
    jersey: "",
    number: "",
    quantity: 1,
  };
  setPlayers([...players, newPlayer]);
};
```

**Process**:
1. Click the green "+" button at the bottom of the player section
2. A new blank player card is added to the roster
3. Unique ID is generated using timestamp
4. Default quantity is set to 1
5. All other fields start empty

**Code Reference**: `src/components/Sidebar.jsx:418-428`

<Tip>
  The "+" button is positioned at the bottom-right of the player section for easy access. You can add unlimited players to your roster.

### 2. Removing Players

Delete players with the red "✕" button on each card:

```javascript
const handleRemove = (id) => {
  setPlayers(players.filter((p) => p.id !== id));
};
```

**Features**:
- **Confirmation**: Click red "✕" to remove
- **Instant update**: Player is immediately removed from roster
- **Filtered state**: Uses array filter to remove by ID
- **No undo**: Deletion is immediate (design can be re-edited)

**Code Reference**: `src/components/Sidebar.jsx:430-432`

<Warning>
  Removing a player deletes all their information. If you save the uniform design after deletion, the player data cannot be recovered.
</Warning>

### 3. Editing Player Details

All player fields support real-time editing:

```javascript
const handleChange = (id, field, value) => {

    players.map((p) => (p.id === id ? { ...p, [field]: value } : p)),

```

**Update Logic**:
- Find player by unique ID
- Update only the changed field
- Preserve all other player data
- Trigger re-render with new state

## Data Processing

When the uniform is saved, player data is processed for order submission:

### Roster Aggregation

Player data is aggregated by size and gender:

```javascript
const tallasMap = {};
players.forEach((p) => {
  const tipo = p.gender === "H" ? "CABALLERO" : 
               p.gender === "M" ? "DAMA" : p.gender;
  const talla = p.jersey || "";
  const key = `${tipo}-${talla}`;
  if (!tallasMap[key]) {
    tallasMap[key] = { tipo, talla, cantidad: 0 };
  }
  tallasMap[key].cantidad += p.quantity || 0;
});
const tallas = Object.values(tallasMap);
```

**Aggregation Logic**:
1. Group players by gender and size combination
2. Sum quantities for each group
3. Generate tallas array for inventory
4. Used for production planning

**Code Reference**: `src/pages/SelectedData.jsx:88-101`

### Individual Player List

Each player's details are preserved in the order:

```javascript
listado: players.map((p) => {
  const item = {
    nombre: p.name || "",
    talla: p.jersey || "",
    cantidad: p.quantity || 0,
    genero: p.gender === "H" ? "CABALLERO" : 
            p.gender === "M" ? "DAMA" : p.gender,
  };
  if (p.number !== undefined && p.number !== null && p.number !== "") {
    item.numero = p.number;
  }
  return item;
})
```

**Features**:
- Complete player information
- Optional number field (only included if provided)
- Gender mapped to Spanish terms
- Empty fields handled with defaults

**Code Reference**: `src/pages/SelectedData.jsx:111-123`

## Error Handling

### Validation

While the UI allows empty fields, best practices include:

- **Name validation**: Ensure players have names before submission
- **Size selection**: Verify all players have jersey sizes
- **Quantity check**: Confirm quantities are greater than 0
- **Number uniqueness**: Consider validating unique jersey numbers

### Event Propagation

Input fields use `stopPropagation` to prevent parent click handlers:

```javascript
onClick={(e) => e.stopPropagation()}
```

**Purpose**:
- Prevents player selection when editing fields
- Allows smooth dropdown and input interaction
- Maintains expected UX behavior
