## Overview

The order process takes your custom uniform design through review, contact information collection, and final submission to the Dark Lion API. This multi-step workflow ensures all design details are correct before sending your order for production.

<Note>
  The order process is managed through the `SelectedData` page, which serves as both a review interface and submission portal.
</Note>

## Order Workflow

The complete order process follows these stages:

<Steps>
  <Step title="Design Completion">
    Complete your uniform customization on the `/personalizar/:modelo` page
    
    - Select all design elements
    - Add team members
    - Choose colors and fonts
    - Upload logo
  </Step>
  
  <Step title="Save Design">
    Click "Guardar Uniforme" to proceed to review
    
    - Design data is saved to navigation state
    - Redirects to `/datos-seleccionados`
    - All customization is preserved
  </Step>
  
  <Step title="Review Design">
    Inspect your complete uniform design
    
    - View all customization choices
    - See player roster
    - Preview uniform visuals
    - Edit if needed
  </Step>
  
  <Step title="Enter Contact Info">
    Provide your contact details
    
    - Name (required)
    - Phone number (required, 10 digits)
    - Email address (required, valid format)
  </Step>
  
  <Step title="Submit Order">
    Send your design to Dark Lion
    
    - Data validated before submission
    - Sent to production API
    - Confirmation message displayed
  </Step>
</Steps>

## Order Submission

When the contact form is submitted, data is sent to the Dark Lion API.

### API Endpoint

```
POST https://api-darklion.onrender.com/api/designs/client-and-design
```

### Payload Structure

The submission payload combines design, player, and contact data:

```json
{
  "name": "Customer Name",
  "email": "customer@email.com",
  "phone_number": 5551234567,
  "id_product": 1,
  "design_file_url": "https://example.com/design.png",
  "tela": "Microfibra",
  "tallas": [
    {
      "tipo": "CABALLERO",
      "talla": "L",
      "cantidad": 5
    }
  ],
  "listado": [
    {
      "nombre": "Player Name",
      "talla": "L",
      "cantidad": 1,
      "genero": "CABALLERO",
      "numero": 10
    }
  ]
}
```

### Payload Preparation

The `preparePayload` function structures all data:

```javascript
const preparePayload = () => {
  // Aggregate tallas by tipo and talla
  const tallasMap = {};
  players.forEach((p) => {
    const tipo =
      p.gender === "H" ? "CABALLERO" : 
      p.gender === "M" ? "DAMA" : p.gender;
    const talla = p.jersey || "";
    const key = `${tipo}-${talla}`;
    if (!tallasMap[key]) {
      tallasMap[key] = { tipo, talla, cantidad: 0 };
    }
    tallasMap[key].cantidad += p.quantity || 0;
  });
  const tallas = Object.values(tallasMap);
  
  return {
    name: contactData.nombre,
    email: contactData.correo,
    phone_number: parseInt(contactData.telefono.replace(/\D/g, "")),
    id_product: 1,
    design_file_url: design.design || "https://example.com/dummy.png",
    tela: design.fabricType || "Default",
    tallas,
    listado: players.map((p) => ({ /* ... */ })),
  };
};
```

**Key Transformations**:
- Phone number converted to integer (digits only)
- Gender codes mapped to Spanish terms (H→CABALLERO, M→DAMA)
- Sizes aggregated by type and size for inventory
- Player numbers only included if provided
- Default values for missing data

**Code Reference**: `src/pages/SelectedData.jsx:88-125`

### Submission Handler

The `handleSendUniform` function manages the submission process:

```javascript
const handleSendUniform = async () => {
  // Validate contact data
  if (!validateContactData()) {
    return;
  }
  
  setSending(true);
  
  const payload = preparePayload();
  console.log("Payload being sent:", payload);
  
  try {
    const res = await fetch(
      "https://api-darklion.onrender.com/api/designs/client-and-design",
      {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(payload),
      },
    );
    
    if (!res.ok) {
      const txt = await res.text();
      throw new Error(`Status ${res.status}: ${txt}`);
    }
    
    const data = await res.json();
    console.log("Respuesta API:", data);
    alert("Uniforme enviado con éxito!");
    setShowContactForm(false);
    
    // Reset form
    setContactData({ nombre: "", telefono: "", correo: "" });
    setFieldErrors({ nombre: "", telefono: "", correo: "" });
  } catch (err) {
    console.error("Error sending uniform:", err);
    setFieldErrors((prev) => ({
      ...prev,
      correo: `Error al enviar: ${err.message}`,
    }));
  } finally {
    setSending(false);
  }
};
```

**Submission Flow**:
1. Validate contact information
2. Set sending state (disables buttons)
3. Prepare payload
4. Log payload for debugging
5. Send POST request to API
6. Check response status
7. Parse JSON response
8. Show success alert
9. Reset form to review view
10. Clear contact data
11. Handle errors by displaying message
12. Always clear sending state

**Code Reference**: `src/pages/SelectedData.jsx:127-169`

<Note>
  The payload is logged to the console before submission, which is helpful for debugging but should be removed in production for security.
</Note>

## Best Practices

<Tip>
  **For a smooth order process**:
  1. Review all design details carefully before proceeding
  2. Verify player roster is complete and accurate
  3. Use a valid email address for order confirmation
  4. Provide a working phone number for contact
  5. Double-check all information before clicking "Contactame"
</Tip>

<Warning>
